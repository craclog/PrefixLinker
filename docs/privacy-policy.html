PrefixLinker — Privacy Policy

Last updated: March 2026

Overview

PrefixLinker is a Chrome extension that converts user-defined prefix patterns (e.g. CSWPR-12345) into clickable links on web pages. This policy explains what data the extension handles and how.

Data We Collect

PrefixLinker collects and stores only the rules you explicitly create:

  • Prefix — the text pattern you want to detect (e.g. CSWPR-)
  • URL Template — the link destination you configure (e.g. https://www.google.com/search?q={match})

No other data is collected. The extension does not read, record, or transmit the content of web pages you visit.

How Your Data Is Stored

Rules are stored exclusively using Chrome's built-in chrome.storage.sync API. This means:

  • Data is saved locally in your browser.
  • If you are signed in to Chrome, Google may sync this data to your other devices through your Google account, subject to Google's Privacy Policy.

PrefixLinker does not operate its own servers. No data is ever sent to the extension developer or any third party.

Permissions Explained

  • storage — required to save and sync your prefix/URL rules across browser sessions and devices.
  • Access to all websites (<all_urls>) — required because prefix patterns can appear on any website. The extension only scans visible text for matching patterns and injects anchor elements for matches. It does not collect, log, or transmit any page content.

Remote Code

PrefixLinker does not load or execute any code from remote servers. All scripts are bundled inside the extension package and reviewed at install time.

Children's Privacy

This extension is not directed at children under 13. It does not knowingly collect any personal information from anyone.
